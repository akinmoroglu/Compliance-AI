Core Use Cases
✈️
Pre-flight Check (Primary)
Advertiser or agency writes an ad. Before submitting to Meta, they run it through Compliance AI. Get violations, severity, and fixes. Launch with confidence.
Write copy → Upload creative → Check → Fix → Submit to Meta
🔴
Rejected Ad Recovery (v3)
Meta rejects an ad with a vague code ("Misleading Claims"). Agency pastes the rejected copy + creative into Compliance AI to pinpoint the exact violation, get the fix, and push the corrected version as a new draft — without touching Meta Ads Manager manually.
Rejection notice → Paste ad → Check → Fix → Apply Fix to Meta (1-click)
🏢
Agency Bulk Audit (v3)
Agency manages dozens of ad accounts. Run Compliance AI across all active creatives for a connected account to catch policy drift before Meta flags the account.
Connect ad account → Fetch active ads → Batch check → Risk report
⚠️
Targeting Risk Advisory
Ad copy for alcohol or gambling product. User provides age targeting. Targeting Checker deterministically validates age_min and country rules — no LLM guesswork. If data is missing, advisory is raised but score is not penalized.
Copy + targeting data → Deterministic check → Hard violation or advisory

Category Classifier
Assigns a single product/service category. Every downstream agent uses this — it's the pipeline's routing decision.
LLM Phase 1 · goroutine 1
Input

Output

Meta's most-enforced rule. Dedicated agent for 3 patterns (Direct Assertion, Rhetorical Question, Audience Address) across 6 protected attribute types. Gray zone rule: when in doubt, flag.

Model routing note: Flash used for v2. If false positive rate is empirically high in testing, upgrade this agent to gemini-1.5-pro (config flag, no code change).
Vision Agent ★ New
Analyzes images or video frames for visual policy violations. Two-part output: first extracts ALL visible text character by character (OCR), then checks for visual violations separately. Multipart Gemini call.
LLM + Vision Phase 1 · goroutine 5
Input
image_bytes: []byte — or — video_frames: [][]byte (scene-change-based, ≤30 frames/clip)
ad_copy: string (context)
category: string
Output (two-part structured JSON)
Part 1 — extracted_text: string (exact OCR of all text in image/frames)
Part 2 — violations[]: { code, explanation, frame_index? }
visual_check_confidence: "high" | "partial" | "low"
image_bytes stored in request context → passed raw to Alignment Agent
Files: vision_agent.go (create) · vision_agent.txt (create)
Upload: multipart form-data (not base64 JSON)
OCR approach: Two-part prompt forces text extraction before visual scan — avoids skipping small disclaimer text
Failure: advisories[] += ADV_VISUAL_PARTIAL · no score penalty
Audio Agent ★ New
Transcribes voiceover/audio from video ads. Voiceovers are the #1 place advertisers embed guaranteed-outcome claims and personal attribute assertions that visual review misses.
LLM + Audio Phase 1 · goroutine 6
Input
audio_bytes: []byte (.mp3 extracted by ffmpeg from video)
Only runs if video input is present
Output
audio_transcript: string (stored in request context)
audio_analysis_status: "transcribed" | "no_audio_track" | "failed" | "not_provided"
Files: audio_agent.go (create) · audio_agent.txt (create)
Why separate from Vision Agent: Different input type, different failure mode, independent testability. Merging into Vision would make prompt + output schema complex.
Transcript is passed to: Violations Agent, Personal Attribute Agent, Alignment Agent
Failure: advisories[] += ADV_AUDIO_NOT_CHECKED · no score penalty
LP Agent ★ New
Fetches and analyzes the landing page. Uses Firecrawl API (not net/http) to handle JS-rendered SPAs, Cloudflare walls, and tracking redirect chains — the majority of agency LP stacks.
HTTP + LLM Phase 1 · goroutine 7
Input
lp_url: string
category: string
ad_copy: string (context for comparison)
Output
lp_violations[]: { code, explanation } — prohibited content found on LP
lp_text_excerpt: first 2000 chars + last 1000 chars + <title>, <meta desc>, all <h1>/<h2> tags
key_claims[]: string[] — claims extracted for Alignment Agent
lp_analysis_status: "fetched" | "failed_timeout" | "failed_js_rendered" | "not_provided"
Files: lp_agent.go (create) · lp_agent.txt (create)
Why Firecrawl vs net/http: ~80-90% of agency LPs (Shopify headless, ClickFunnels, Unbounce) are JS-rendered. Plain HTTP GET returns blank body. Agencies will stop trusting the tool if LP check fails constantly.
Timeout: 8 seconds (not 3)
Future: migrate to self-hosted Playwright for cost control
Targeting Checker ★ New
Pure deterministic Go. No LLM. Compares age_min and countries[] against category-specific rules. Runs in <1ms. Produces hard violations when rules are broken, advisories when data is missing.
Deterministic Go Phase 1 · goroutine 8
Input
category: string
age_min: int (-1 = not provided)
countries: []string (empty = not provided)
Output
targeting_violations[]: { code, explanation } — hard breaches
advisories[]: { code, message, category } — data missing for restricted category
Advisory escalation rules
Age escalation (7 categories): Alcohol, Gambling, Online Pharmacies, Tobacco, Cannabis, Dating, Sexual Wellness — missing age_min → advisory upgrades to soft violation when copy contains explicit category signals
Geo advisory only (4 categories): Online Gambling, CBD, Crypto Exchange, Firearms Accessories
File: targeting_checker.go (create)
Hard breach example: category=Gambling + age_min=16 → R_TARGETING_AGE_BREACH (violation, not advisory)
Advisory example: category=Alcohol + age_min=-1 → ADV_MISSING_AGE_TARGETING
Alignment Agent ★ New — Phase 2
Checks cross-source consistency. Detects direct contradictions between ad copy, visual, audio, and LP — not natural variation. Only runs when 2+ sources are present.
Phase 2 — Conditional LLM + Vision
Input — raw sources, not summaries
ad_copy: string
image_bytes: []byte — raw image from request context (NOT a description — avoids "game of telephone" lossy summary)
audio_transcript: string (from Audio Agent via context)
lp_text_excerpt: string + key_claims[] (from LP Agent via context)
existing_violation_codes: []string (to avoid double-flagging)
Output
alignment_violations[]: { code, explanation, source_a, source_b }
4 codes: ALIGN_PRODUCT_MISMATCH · ALIGN_CLAIM_MISMATCH · ALIGN_PRICE_MISMATCH · ALIGN_VISUAL_COPY_MISMATCH
NOT a violation — examples
Copy shorter than LP (normal) · Same product in different colors · Price range that includes stated price
Files: alignment_agent.go (create) · alignment_agent.txt (create)
Trigger: 2+ of: ad_copy, image_bytes, audio_transcript, lp_text present after Phase 1
Scoring: ALIGN_ codes → M_ tier (Rule 4) — serious but not structural category violation
Failure: skip gracefully — no advisory raised
Fix Generation Agent
Generates actionable remediation per violation. P_ = state prohibition clearly. A_ = explain auth path, link Meta resources. R_/M_ = concrete rewrite (original phrase → replacement).
LLM Phase 3 · always runs
Input
ad_copy: string
all_violations[]: merged from Phase 1 + Phase 2
category: string
Output
fixes[]: { violation_code, fix_type, explanation, rewritten_copy? }
v3 planned enhancement
Multiple tone variants: compliance_safe · marketing_persuasive · brand_matched
Files: fix_generation.go · fix_generation_agent.txt
Failure: returns empty fixes[] — violations still shown, fix section shows "unavailable"

Deterministic waterfall. No LLM. Same input always produces the same score — no variance. Reads all violation codes and applies 6-rule cascade.

Output

Advisory System — What It Is & How It Renders

Advisories are soft signals — things the system couldn't fully verify because data was missing or a check failed. They do not affect the compliance score. They render below the violations section, visually distinct. The goal: surface risk without punishing the advertiser for incomplete inputs.

Tech Stack
LayerTechnologyDecision & Rationale
LLM gemini-flash-latest Single model for all LLM agents in v2. Fast, cheap, multimodal, structured output via ResponseSchema. Model routing config flag per agent is built-in — upgrade individual agents to Pro if empirical false-positive rates justify it. No cross-vendor mixing in v2.
Vision + Audio Gemini multipart Same model — handles image bytes and audio natively. No additional vendor. Vision Agent: image bytes as genai.Blob. Audio Agent: .mp3 bytes from ffmpeg as genai.Blob.
Video pre-processing ffmpeg via exec.Command Server-side. Extracts: (1) video frames — scene-change-based, ≤30 per clip; (2) audio track as .mp3. Both in memory — no orphaned temp files.
LP fetch Firecrawl API Handles JS-SPAs, Cloudflare, tracking redirects. ~80-90% of agency LP stacks are JS-rendered — net/http fails on most of them. 8s timeout. Migration to self-hosted Playwright planned for cost control.
File upload Multipart form-data Replaces JSON-only handler. Required for image/video file uploads. Native Go support.
Concurrency Go goroutines + sync.WaitGroup 8 Phase 1 agents run in parallel. Fault-tolerant — individual agent errors are captured per-goroutine, do not crash the pipeline.
Deterministic logic Pure Go functions Targeting Checker + Scoring Engine. Zero LLM variance. Same input = same output, always.
Backend Go + Encore.dev + GCP Existing stack. No infrastructure changes for v2.
Frontend Vue 3 + TypeScript + Vite + Tailwind v4 New components: AdvisoryPanel.vue · file upload fields in Step 2 · targeting input (age + country selector)
Meta API (v3) Rockads existing integration Apply Fix uses the Meta Marketing API already integrated in the Rockads core platform. No new API client needed — compliance checker returns suggested_copy, UI calls existing Rockads Meta endpoint to push as new ad draft.

What v2 Adds over v1

Coverage gaps closed

  • 👁Visual violations detected: v1 was copy-only — blind to images and video.
  • 🎙Audio/voiceover analyzed: #1 source of hidden guaranteed-outcome and personal attribute claims. Completely missed in v1.
  • 🔗Landing page scanned: v1 ignored what the user lands on. LP can contain prohibited content that causes rejection even if copy is clean.
  • ⚖️Cross-source alignment: v1 had no concept of copy ↔ visual ↔ LP consistency. Alignment Agent closes this.
  • 🎯Targeting validation: v1 had zero targeting awareness. Agencies can now get age/geo compliance checks.
  • 📋27 violation codes vs ~15: Full Meta policy coverage including crisis exploitation, misinformation, vaccine discouragement, skin whitening, and more.
  • 🌍Multilingual: v1 relied on English keyword patterns. v2 agents detect semantically equivalent violations in any language.

Architecture improvements

  • 🏗8 agents, each single-responsibility: Audio and Vision are separate; Personal Attribute is separate from Violations. Easier to test, improve, and swap individually.
  • ⚡True parallel Phase 1: 8 goroutines. Slowest agent determines phase time — not the sum.
  • 🔀Conditional Phase 2: Alignment Agent only runs when relevant. No wasted LLM call for copy-only checks.
  • 🛡Fault isolation: One agent failing = partial result, not 500 error.
  • 📊Deterministic scoring + targeting: Zero LLM variance in the final verdict.
  • 🔒Structured output enforced: ResponseSchema on every LLM agent — no free-text hallucinations in JSON fields.
  • 📸Raw image to Alignment Agent: Image bytes passed via request context — not a lossy text description. No "game of telephone."

Known Concerns & Risks

Technical risks

  • ⏱Latency: 8 parallel goroutines, but Vision + Audio + LP are slowest (file bytes + HTTP). Target P99 under 10 seconds. Needs benchmarking — not assumed.
  • 🎥Video frame sampling: Scene-change extraction is heuristic. Short scenes (<1s cuts in fast-editing ads) may be missed. Empirical tuning needed — frame count is currently "≤30" but this is a starting point.
  • 🌐JS-rendered LPs: Firecrawl handles most cases. But some LP stacks (paywall-gated, cookie-consent-blocked, login-required) will still fail. Advisory always shown when LP fetch fails.
  • 💾Large video memory: Frame extraction + audio bytes in-memory for long clips. Need max-duration guard. Engineering decision on limit.
  • 🏷Logo/celebrity detection: Gemini Vision unreliable for non-major brand logos and faces. Copyright/endorsement violations remain best-effort. UI must label detection confidence explicitly.

Product / accuracy risks

  • 🔀Alignment false positives: Natural variation between copy and LP (copy is always shorter, more persuasive) could fire ALIGN_ incorrectly if prompt examples aren't precise enough. Prompt requires extensive NOT-a-violation examples and controlled testing before launch.
  • ❓Advisory fatigue: Too many advisories = users ignore all of them. Must limit surface area — only raise when category is explicitly restricted AND the specific input type was expected.
  • 📋Policy drift: Meta updates ad policies frequently. Prompt files are static — they will go stale without a review process. Quarterly minimum recommended.
  • 🧠Personal Attribute false positives: "Are you looking for a gym?" should NOT flag PA-004 (age). Flash is capable but borderline cases need a labeled test set to measure before shipping. Model upgrade path (gemini-1.5-pro) is available via config flag if needed.
  • 📉Score gaming: Deterministic scoring is transparent. Sophisticated users will learn to write M_ instead of R_ by minor wording changes. Unavoidable — monitor in production.
